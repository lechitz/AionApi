# Standards (aion-api)

This folder defines the coding and documentation standards used across the aion-api codebase. These files describe **how we write**, **name**, and **structure** code and tests.

## What You Will Find Here

- `file-naming-conventions.md`  File structure and naming patterns
- `naming.md`                   Variable and function naming rules
- `errors.md`                   Error handling conventions
- `logging.md`                  Logging and observability patterns
- `testing.md`                  Test structure and coverage rules

## How To Use

1) Check these standards before writing or refactoring code.
2) Use the examples as templates.
3) Keep changes consistent with the `category` context, the reference implementation.

## Principles

- Consistency over preference.
- Small, readable units of code.
- Observability at boundaries.
- Semantic errors and clear contracts.
